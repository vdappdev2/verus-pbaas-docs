# getoffers

> Get all open offers for a specific currency or identity.

**Category:** Marketplace
**Daemon help:** `verus help getoffers`

---

## Summary

Returns buy and sell offers for a given currency or identity — both offers *of* the asset and offers *for* the asset. This is the primary discovery tool for the on-chain marketplace.

---

## Syntax

```
getoffers "currencyorid" (iscurrency) (withtx)
```

---

## Parameters

| # | Name | Type | Required | Default | Description |
|---|------|------|----------|---------|-------------|
| 1 | `currencyorid` | string | Yes | — | Currency name/i-address or identity name/i-address to query. Must match the `iscurrency` flag — passing an identity name with `iscurrency: true` returns an error. |
| 2 | `iscurrency` | boolean | No | `false` | `true` for currency offers, `false` for identity offers. Strict: cannot mix identity names with `iscurrency: true` or currency names with `iscurrency: false`. |
| 3 | `withtx` | boolean | No | `false` | Include serialized hex of the exchange transaction. Useful for inspecting offer details or passing to `takeoffer` via the `tx` parameter. |

---

## Return value

An object with **dynamic keys** generated by the daemon based on the queried asset. Parse keys dynamically — do not hardcode them.

### Key naming patterns

| Pattern | Meaning |
|---------|---------|
| `currency_{id}_offers_in_currency_{id}` | Currency-for-currency offers |
| `ids_for_currency_{id}` | Identities offered for sale for that currency |
| `currency_{id}_for_ids` | Currency offered to buy identities |
| `ids_for_id_{id}` | Identity-for-identity swap offers |

Each key maps to an array of offer objects.

### Currency offer object

| Field | Type | Description |
|-------|------|-------------|
| `currencyid` | string | i-address of the currency being offered or requested |
| `price` | number | Price per unit |
| `offer` | object | Offer details (see inner offer below) |

### Identity offer object

| Field | Type | Description |
|-------|------|-------------|
| `identityid` | string | i-address of the identity being offered or requested |
| `price` | number | Price in the paired currency |
| `offer` | object | Offer details (see inner offer below) |

### Inner offer detail

| Field | Type | Description |
|-------|------|-------------|
| `offer` | object | What the maker is offering — currency map (`{"i-address": amount}`) and/or identity info (`{"name": "...", "identityid": "...", "systemid": "...", "original": number}`) |
| `accept` | object | What the maker wants in return — same structure |
| `blockexpiry` | number | Block height at which this offer expires |
| `txid` | string | Transaction ID of the offer — pass to [`takeoffer`](takeoffer.md) to accept |

### Edge cases

- **No offers:** Returns empty object `{}`. Not an error.
- **Wrong `iscurrency` flag:** Identity name with `iscurrency: true` returns "Currency specified as source is not valid". Currency name with `iscurrency: false` returns "Identity specified as source is not valid".
